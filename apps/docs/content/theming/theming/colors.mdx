# Colors

Colors define the Extraction UI palette and provide consistent usage across components.

## Overview

Each color has 11 shades:

- **50-100** — Very light: use for backgrounds, disabled states
- **200-400** — Light: borders, hover states
- **500-600** — Mid-tone: primary text, icons
- **700-800** — Dark: secondary text, emphasis
- **900-950** — Very dark: text contrast, prominent emphasis

## Accessibility

All color combinations are tested to meet **WCAG AA** (minimum 4.5:1 contrast) for text and **WCAG AAA** (7:1) where possible. [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/)

## Customizing Colors

The recommended way to customize Extraction UI palette is by overriding the custom theme CSS variables.

## Basic override

Add an override in your main CSS file, this will also generate all shades automatically:

```sass filename="main.css"
:root {
  --color-primary: oklch(0.5 0.22 275);
  --color-accent: oklch(0.5 0.22 280);
  --color-neutral: oklch(0.5 0.03 275);
}
```

## Advanced override

For complete control over all shades and their foreground colors, use the advanced approach. You don't have to use the color-mix formulas. You can set any color you want for each shade and its foreground.


## Creating Palettes

To create a dynamic palette to be used in all components, add a new utility in your main CSS file:

```sass filename="main.css"

  --palette: var(--color-new);
  --palette-foreground: var(--color-new-foreground);

  --palette-50: var(--color-new-50);
  --palette-50-foreground: var(--color-new-50-foreground);
```
